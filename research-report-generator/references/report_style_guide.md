# Report Style Guide

This reference provides detailed writing conventions and formatting standards for research reports generated by this skill.

## Document Structure

```
# [Report Title]

> [One-line description of what this report covers, including time scope and data sources]

---

## Executive Summary

### Chapter 1: [Title]
- Key finding 1
- Key finding 2

### Chapter 2: [Title]
- Key finding 1
- Key finding 2

...

---

## Chapter 1: [Title]

**Core Findings:**
- **Finding 1**: [Concise statement with key number]
- **Finding 2**: [Concise statement with key number]
- **Finding 3**: [Concise statement with key number]

### 1.1 [Subsection]

[Detailed analysis with data tables, citations, and cross-references]

### 1.2 [Subsection]

...

---

## Chapter 2: [Title]

...

---

## Appendix (optional)

### Data Sources
| # | File Name | Description | Date |
|---|-----------|-------------|------|
| 1 | ...       | ...         | ...  |

### Methodology Notes
...

### Glossary
...
```

## Citation Formats

### From Spreadsheets (Chinese)
```
（来源：[文件名], Sheet "[工作表名]", 第X行）
（来源：Switch 2 BOM Estimate 2026 03 03.xlsx, Sheet "BOM Estimate"）
```

### From Spreadsheets (English)
```
(Source: [filename], Sheet "[sheet name]", Row X)
(Source: Switch 2 BOM Estimate 2026 03 03.xlsx, Sheet "BOM Estimate")
```

### From Documents / Interviews (Chinese)
```
（来源：[文件名], [专家名/章节名]）
（来源：Combined Memo of Nintendo Switch 2.docx, 富士康专家访谈）
```

### From Documents / Interviews (English)
```
(Source: [filename], [expert name / section title])
(Source: Combined Memo of Nintendo Switch 2.docx, Foxconn Expert Interview)
```

### From Web Sources (Chinese)
```
（来源：[媒体名], [日期]）
（来源：日经新闻, 2026年3月15日）
```

### From Web Sources (English)
```
(Source: [media name], [date])
(Source: Nikkei Asia, March 15, 2026)
```

## Table Formatting

### Financial Data Tables
- Right-align numeric columns
- Use consistent decimal places (typically 1-2 for percentages, 0-2 for currency)
- Include units in column headers, not in every cell
- Use `—` for missing data, not blank cells
- Bold key rows (totals, targets, thresholds)

### Comparison Tables
- Place the most important dimension in the leftmost column
- Sort by the most relevant metric (descending by default)
- Highlight outliers or key values with bold

## Tone and Voice

- **Objective**: Present facts and analysis, not opinions. When interpretation is needed, clearly label it.
- **Precise**: "Revenue increased 16.7% YoY to ¥240.56B" not "Revenue grew significantly"
- **Structured**: Use numbered lists for sequential items, bullet points for parallel items
- **Professional**: Avoid colloquialisms, emoji, or casual language in the report body
- **Balanced**: Present risks alongside opportunities, bear cases alongside bull cases

## Common Pitfalls to Avoid

1. **Vague quantification**: Never write "significant" / "substantial" / "大幅" without a number
2. **Unsourced claims**: Every data point needs a citation
3. **Mixing time periods**: Clearly label whether figures are quarterly, annual, TTM, etc.
4. **Currency confusion**: Specify currency (USD, JPY, CNY, HKD) on first mention
5. **Survivorship bias**: When analyzing trends, note any data gaps or missing periods
6. **Correlation vs causation**: When discussing relationships, be precise about what the data shows vs. what is inferred
